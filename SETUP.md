# 🚀 RAG Pipeline - Setup Guide

Complete guide to run the full-stack RAG application with FastAPI backend and React frontend.

---

## 📋 Prerequisites

- **Python 3.9+** (for backend)
- **Node.js 18+** (for frontend)
- **Git** (optional)

---

## 🛠️ Installation Steps

### 1. Install Backend Dependencies

```bash
# Activate your virtual environment (if using one)
# Windows
.venv\Scripts\activate

# Linux/Mac
source .venv/bin/activate

# Install Python packages
pip install -r requirements.txt
```

### 2. Install Frontend Dependencies

```bash
cd frontend
npm install
cd ..
```

### 3. Configure Environment Variables

Create a `.env` file in the root directory:

```bash
GROQ_API_KEY=your_groq_api_key_here
```

Get your free API key from: https://console.groq.com/

---

## ▶️ Running the Application

You need to run **both** backend and frontend in **separate terminals**.

### Option 1: Using Run Scripts (Recommended)

**Terminal 1 - Backend:**
```bash
# Windows
run-backend.bat

# Linux/Mac
chmod +x run-backend.sh
./run-backend.sh
```

**Terminal 2 - Frontend:**
```bash
# Windows
run-frontend.bat

# Linux/Mac
chmod +x run-frontend.sh
./run-frontend.sh
```

### Option 2: Manual Commands

**Terminal 1 - Backend (Port 8000):**
```bash
python -m uvicorn backend.main:app --reload --port 8000
```

**Terminal 2 - Frontend (Port 3000):**
```bash
cd frontend
npm run dev
```

## 🌐 Access the Application

Once both servers are running:

- **Frontend UI**: http://localhost:3000
- **Backend API**: http://localhost:8000
- **API Documentation (Swagger)**: http://localhost:8000/api/docs
- **Alternative API Docs (ReDoc)**: http://localhost:8000/api/redoc

## 📚 Usage Guide

### 1. Upload Documents

Click on the upload area in the UI to add new documents:
- Supported formats: PDF, TXT, CSV, Excel, Word, JSON
- Files are automatically processed and indexed
- Check the success message for confirmation

### 2. Ask Questions

Type your question in the input box and press Enter or click Send:
- Example: "What is DBMS?"
- Example: "Explain ACID properties"
- Example: "What is an operating system?"

### 3. View Sources

Each AI response includes:
- **Answer**: Generated by LLM based on your documents
- **Sources**: Referenced documents with page numbers
- **Confidence Score**: Match percentage for each source
- **Query Time**: Response time in seconds

### 4. Adjust Settings

Use the Top-K selector to control how many document chunks to retrieve:
- **1-3**: Fast, focused answers
- **5-7**: More comprehensive answers
- **10**: Maximum context (slower)

## 🐛 Troubleshooting

### Backend Issues

**Error: "No module named 'backend'"**
```bash
# Make sure you're in the project root directory
cd d:\ML Projects\RAG
python -m uvicorn backend.main:app --reload
```

**Error: "GROQ_API_KEY not found"**
```bash
# Check your .env file exists in the root directory
# Verify it contains: GROQ_API_KEY=your_key
```

**Error: "FAISS index not found"**
```bash
# Your vector store needs to be built first
# Run the original app.py to build it:
python app.py
```

### Frontend Issues

**Error: "Cannot connect to backend"**
- Make sure backend is running on port 8000
- Check CORS settings in [backend/main.py](backend/main.py)
- Try: http://localhost:3000 instead of 127.0.0.1

**Error: "npm install fails"**
```bash
# Clear npm cache and retry
npm cache clean --force
cd frontend
rm -rf node_modules package-lock.json
npm install
```

**Port already in use**
```bash
# Change port in vite.config.js
# Or kill the process using the port
# Windows: netstat -ano | findstr :3000
# Linux: lsof -ti:3000 | xargs kill
```

## 🔐 Security Notes

- **Never commit `.env` file** (already in .gitignore)
- **Uploaded files** are stored in `data/uploaded/` (ignored by git)
- **Vector store** is ignored by git (regenerate on deployment)
- **Regenerate API keys** if accidentally committed to git
